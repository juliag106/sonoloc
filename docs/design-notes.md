# 设计说明

这里记录一些实现取舍和背后的原因，方便后续维护与扩展。

## 为什么以 NumPy 为核心、PyTorch 可选

SELD 的经典基线（GCC-PHAT、SRP-PHAT、MUSIC、声强向量、各类指标）都能用纯
NumPy/SciPy 精确实现，安装轻、易复现、便于在 CI 上快速跑通。神经网络只是其中一条
路线，因此放进可选的 `[torch]` extra：没有 GPU 或只想用经典方法的用户零负担。

## DOA 的符号约定

- 远场平面波在第 `m` 个麦克风的传播时延为 `tau_m = -(p_m · u) / c`，`u` 为指向
  声源的单位向量。朝声源方向偏移的麦克风更早接收，时延为负。
- MUSIC 伪谱使用 `a^H · E_noise`：导向向量必须取共轭转置，否则会在方位角上
  出现 180° 的前后镜像（这个坑在早期实现里踩过，见相关修复提交）。
- FOA 声强向量采用常见的 Ambisonics 声源编码 `[X, Y, Z] = u · W`，此时有源声强
  指向声源，DOA 与声强方向一致，不需要再取负号。

## 弱标注与池化

只有片段级标签时，池化函数决定了梯度如何回传到帧级：`max` 只更新单帧、容易漏检
短事件；`mean` 会稀释强峰。线性/指数 softmax 用预测值自加权，实践中通常落在两者
之间，是较稳的默认选择。

## 合成数据的边界

当前渲染是**自由场平面波**近似，不含房间混响。它足以支撑单元测试、演示和方法对拍，
但与真实混响、近场、运动声源仍有差距。后续计划接入镜像法 / RIR 卷积来补齐这一环
（代码里以 `TODO` 标注了接入点）。

## 空间混叠

四面体阵列半径约 4.2 cm，空间奈奎斯特频率约 2.5 kHz。低频段所有方向的导向向量都
接近全 1，方位可分性差；因此定位应更依赖中高频，仿真里也会对源信号做带限处理。

## 已知取舍 / 待办

- 单-ACCDOA 每类只保留一个方向，重叠同类事件需要 multi-ACCDOA + 置换不变训练。
- 指标是段级的简化实现，未完全复刻 `sed_eval` 的全部选项。
- 训练循环尚未内置，`models` 目前只提供网络结构与损失。
